Inheritance

Two decades ago when Java was born, it brought a concept of Inheritance. What is inheritance and what are the rules of inheritance?

Whether you are passing a primitive variable or the reference to an object; you are always passing a copy of bits in the variable. The called method then gets its own copy of the reference variable to do with it what it likes.But because two identical reference variables refer to the exact same object, if the called method modifies the object (by invoking setter methods, for example), the caller will see that the object the caller's original variable refers to has also been changed. Inheritance allows a class to be a subclass of a superclass and thereby inherit public and protected variables and methods of the superclass. Inheritance is a key concept that underlies IS-A, polymorphism, overriding, overloading, and casting. All classes (except class Object) are subclasses of type Object, and therefore they inherit Object's methods.

Rule: Subclasses know their parent (super classes) that they inherit from but super classes doesn't know anything about the sub classes.
